# Working principles

A few habits that make every prompt in this library more useful.

## Always customize

The placeholders are the point. Generic inputs produce generic outputs. Replace every `{{PLACEHOLDER}}` with your specifics: product context, target customer, real user quotes if you have them.

A prompt run on your actual context is worth ten prompts run on hypotheticals.

## Iterate on outputs

Treat the first answer as a draft, not a finished answer. The first output is the model showing you what it understood from your prompt. Read it, find what's wrong or missing, and respond with a sharper version.

Iteration is where the quality lives.

## Validate with real users

AI-generated insights are starting hypotheses, not findings. Test them against real people and real data before committing.

The model can write you a perfect persona, a perfect feature list, a perfect headline. Whether any of it survives contact with your actual customers is an empirical question.

## Use prompts as starting points

A prompt is a way to leave the blank page faster, not a substitute for thinking. The model can give you 10 feature ideas; deciding which 3 to prototype is still your job.

If the answer feels generic, that's because the prompt was generic. Add specifics, then run it again.
